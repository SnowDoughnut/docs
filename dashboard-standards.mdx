description: "How dashboards should be structured and named so KPIs are reported consistently."
---

Standards for how reporting dashboards are structured and named, so the [KPIs](/kpi-definitions) they surface stay consistent across teams and tools.

See [report writing concepts](/report-writing-concepts#dashboards-answer-a-question-in-under-30-seconds) for how to scope a dashboard, lay it out, and decide what earns a place on it. A dashboard reads from the same [flat file](/flat-file-template) every report is built from, and every number on it should trace back to a definition in the [data dictionary](/marketing-reporting-data-dictionary).
