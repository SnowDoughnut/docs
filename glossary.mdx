title: "Glossary"
---

Terms are listed alphabetically. Where a term has its own full page, this entry gives the short version and links out.

### Ad group

Google's term for a group of ads that share the same customer profiles, keywords, and bids within a campaign. Equivalent to a Meta [ad set](#ad-set). See [ad set / ad group / variant naming conventions](/ad-set-group-naming-conventions).

### Ad set

Meta's term for a group of ads that share the same customer profile, budget, schedule, and placements within a campaign. Equivalent to a Google [ad group](#ad-group). See [ad set / ad group / variant naming conventions](/ad-set-group-naming-conventions).

### Campaign

The top-level container for ads in a platform account: it sets the objective, budget type, and funnel position that everything underneath it inherits. See [campaign naming conventions](/campaign-naming-conventions).

### Creative brief

The fill-in workbook that defines what a piece of creative needs to say and do before production starts: objective, customer profile, offer, and format. See [creative brief](/creative-brief) and the [creative brief process](/creative-brief-process). Use "creative briefs" only when referring to more than one, such as the [creative brief library](/creative-briefs).

### Creator

The person who produces the creative work. This is the designer, art director, photographer, videographer, illustrator, copywriter, or other maker who takes the brief and turns it into the actual asset.

### Customer profile

A defined segment of people an ad, campaign, or landing page is built for, described by shared traits, behavior, or list membership. Use "customer profile" for this concept everywhere — not "audience." See [customer profile brief](/customer-profile-brief), [customer profile naming conventions](/customer-profile-naming-conventions), and the [customer profile library](/customer-profile-library-overview).

### Dictionary

The table of approved values for a naming identifier (for example, every valid objective or offer code), built once per category and reused in every name assembled from it. Each [naming convention](#naming-convention) doc holds the dictionary for its level.

### Experiment ID

The short, structured identifier assigned to a test when its hypothesis is approved, appended to the names of everything that belongs to that test. See the [experiment ID SOP](/experiment-ids) and [experiment ID workbook](/experiment-id).

### KPI

Key performance indicator: a specific, named metric a campaign or test is measured against, defined once so everyone reports it the same way (for example, CPA or ROAS). "Metric" is the general word for any measured number; "KPI" is reserved for the metrics defined in [KPI definitions](/kpi-definitions).

### Name jamming

Packing multiple identifiers into a name using their short form instead of their long form, to stay within a platform's character limit. See [naming conventions overview](/naming-conventions-overview#name-jamming-short-identifiers).

### Naming convention

The rules that define how campaign, ad set/ad group, and ad names are built from a shared dictionary of identifiers, so names stay predictable and filterable at scale. Start at the [naming conventions overview](/naming-conventions-overview).

### Owner

The one person accountable for a piece of work from start to finish. The owner drives it forward, makes the calls it needs, coordinates anyone else involved, and answers for the result. Ownership is singular: one named person, not a team or a role, so there's never a question of who to ask.

### Pacing

Tracking spend against a budget over time to confirm a campaign will land on-target by the end of its flight, and adjusting before it over- or under-spends. See [budget pacing](/budget-pacing).

### Peeking

Checking a running test repeatedly and stopping it the moment the result looks significant. Each extra look gives the result another chance to cross the threshold by luck, so stopping on a good day inflates the false-positive rate: the test reports a winner that isn't real. Avoid it by setting the sample size and end date before launch and letting the test run to that plan, whatever the numbers do along the way. Extending is only valid for an external reason (a traffic drop, a tracking break, a seasonal shift), never because the result hasn't reached significance yet.

### SOP

Standard operating procedure: a documented, repeatable process for a recurring task, kept in the [Processes](/campaign-brief-process) tab.

### Standard identifier

A permanent, always-present piece of a name — channel, objective, customer profile, offer, and so on — as opposed to a [testing identifier](#testing-identifier). See [naming conventions overview](/naming-conventions-overview#how-a-name-is-built).

### Statistical significance

The confidence level that a test result reflects a real difference rather than random variation. See the full [statistical significance](/statistical-significance) guide.

### Testing identifier

An identifier appended to a name only while a hypothesis is actively being tested, marking which campaigns, ad sets, or ads belong to that test. See [naming conventions overview](/naming-conventions-overview#how-a-name-is-built) and the [experiment ID SOP](/experiment-ids).

### UTM parameter

A tag appended to a URL that records where a click came from, so traffic and conversions can be attributed back to a campaign, source, and medium. Use "UTM parameter," not "UTM tag." See [UTM parameters](/utm-parameters) and [UTM standards](/utm-standards).
