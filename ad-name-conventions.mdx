## Standard identifiers

Creative name is freeform, for example `AutoGlaze`, and doesn't need a table of its own; every creative has a unique name. Whenever possible, the creative name should match the file name. [See File naming conventions](/file-naming-conventions) for more information.

<Accordion title="Concept review: ad-level identifiers">
  Creative name, together with ad type, offer, CTA, and destination, is what lets you run creative analysis. When two ads are identical except for one field, for example the same creative and CTA but a different offer, the naming convention is what makes that comparison visible in a report without having to open each ad individually. Creative name is the one freeform field here; it doesn't need a table of its own, since every creative has a unique name. Ad type, offer, CTA, and destination are the enumerable ones worth standardizing and logging above.
</Accordion>

## Testing identifiers

<Tip>
  Optional: append only to ads that are part of an active test.
</Tip>

Testing identifiers mark a hypothesis you're currently running, for example a creative format test or an offer test. Because a test only applies to some ads for a limited time, testing identifiers aren't part of the standard identifier set: append one to the end of the ad name, after the standard identifiers, instead of building it into a table above.

- With a testing identifier appended: `AutoGlaze-STATIC-FREETRIAL-TRY-LP-TEST01`

This document only covers where a testing identifier sits in the name. It doesn't cover how an [experiment ID](/glossary#experiment-id) gets created, how long it stays on an ad, or where the result gets logged once the test ends. Visit the [Experiment ID SOP](/experiment-ids) for that.
